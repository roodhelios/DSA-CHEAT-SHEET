Section 01
Variables & Types
🧠 Mental Model

Python variables are labels on sticky notes, not boxes. A label can move to any value — a number today, a string tomorrow. You never declare a type because Python figures it out when you assign. Multiple labels can point to the same note.

💡Think of it like JavaScript's let/var but without any type coercion nonsense. Python just works.
Dynamic Typing

Variables get their type from the value assigned, not a declaration. n = 0 makes n an int. n = "hello" makes it a string. Same variable, different type.

Multiple Assignment & Swap

Python's killer trick: a, b = 1, 2 assigns both at once. And a, b = b, a swaps without a temp variable — Python evaluates the right side fully first.

None — Python's null

None is Python's null. Use if x is None (identity check) instead of if x == None. Both work, but is is the idiomatic way.

⚠️No ++ operator! Python doesn't have ++ or --. Use n += 1 and n -= 1 instead. Writing n++ is a syntax error.
✅Types you'll use: int, float, str, bool, None. Check with type(x) or test with isinstance(x, int).

Section 02
If / Else
🧠 Mental Model

Python uses indentation as syntax — no curly braces. The elif keyword replaces else if. Conditions are plain English: and, or, not instead of &&, ||, !.

elif, not else if

Python's keyword is elif, a contraction of "else if". A chain of if / elif / elif / else checks top-to-bottom and stops at the first true condition.

Chained Comparisons

Python supports 1 < x < 10 — a chained comparison that reads exactly like math. No need to write x > 1 and x < 10.

Truthiness & Falsiness

Falsy values: False, 0, "", [], {}, None. Everything else is truthy. So if my_list: checks if the list is non-empty.

💡Multi-line conditions can be wrapped in parentheses: if (a == 1 and b == 2 and c == 3): — the parens let you break across lines.

Section 04
Math & Division
🧠 Mental Model

Python 3 changed division forever: / always gives a float. Use // for integer floor division. The floor division catches people off guard with negative numbers — it floors toward negative infinity, not toward zero.

⚠️Critical gotcha: -3 // 2 is -2 in Python (floors to -∞). In Java/C++ it's -1 (truncates toward 0). Use int(-3 / 2) to get Java-style truncation.
Division operators

5 / 2 → 2.5 (always float). 5 // 2 → 2 (floor integer). 5 % 2 → 1 (modulo/remainder).

Infinity

float('inf') and float('-inf') represent positive and negative infinity. Useful as sentinel values: initialize a min-tracker to float('inf').

math module

import math gives you math.floor(), math.ceil(), math.sqrt(), math.log(), math.pow(), and constants math.pi, math.e.

Section 05
Lists
🧠 Mental Model

Python lists are dynamic arrays. append and pop from the end are O(1). insert and pop(0) are O(n) because they shift elements. Negative indexing ([-1]) accesses from the end.

⚠️2D list trap! [[0]*4]*4 creates 4 references to the same inner list. Use [[0]*4 for _ in range(4)] to create independent rows.

Core operations & complexities

append(x) O(1) · pop() O(1) · insert(i,x) O(n) · pop(0) O(n) · len(lst) O(1) · lst[i] O(1)

Slicing

lst[start:stop:step] — all optional. lst[::-1] reverses a list. lst[1:4] gives indices 1,2,3. Slicing creates a shallow copy.

enumerate & zip

enumerate(lst) gives (index, value) pairs. zip(a, b) pairs elements from two lists. Both are lazy iterators — wrap in list() to print.

List Comprehension

[x*2 for x in range(5)] creates a new list. Add a filter: [x for x in lst if x > 0]. More readable and faster than equivalent loops.

Section 06
Strings
🧠 Mental Model

Strings in Python are immutable sequences of characters. You can slice and index them like lists, but you can't change a character in place. Building strings with += in a loop is O(n²) — use join() instead.

Immutability

You cannot do s[0] = 'X' — strings are immutable. Convert to a list to mutate, then join back: chars = list(s); chars[0] = 'X'; s = ''.join(chars)

ord() and chr()

ord('a') → 97 (ASCII value). chr(97) → 'a'. Essential for problems involving character arithmetic, like checking if a char is alphabetic or computing offsets.

join() — the right way to build strings

''.join(list_of_chars) builds a string from parts in O(n). Concatenating with += in a loop creates a new string each iteration → O(n²).

💡Useful methods: .split(), .strip(), .lower(), .upper(), .replace(old, new), .startswith(), .endswith().

Section 07
Queues (deque)
🧠 Mental Model

A queue is a double-ended line. Python's collections.deque is a doubly-linked list that supports O(1) append/pop from both ends. Never use a regular list as a queue — list.pop(0) is O(n) and will TLE.

deque vs list for queue ops
DEQUE — O(1) both ends
appendleft(x) ← front append(x) → back popleft() ← front O(1) pop() → back O(1)
LIST — O(n) at front
insert(0, x) ← O(n)! append(x) → O(1) pop(0) ← O(n)! pop() → O(1)
⚠️In BFS, every node gets enqueued once. If dequeue is O(n), your BFS is accidentally O(n²). Always use deque and popleft().

Section 08
Hash Sets
🧠 Mental Model

A set is a hash table with only keys, no values. It gives you O(1) membership testing, insertion, and deletion. No duplicates allowed. Use it when you need "have I seen this before?" in O(1).

Core operations — all O(1)

s.add(x) · s.remove(x) (raises if missing) · s.discard(x) (safe remove) · x in s (O(1) lookup!) · len(s)

Set operations

a & b intersection · a | b union · a - b difference · a ^ b symmetric difference. Or use methods: .intersection(), .union(), etc.

💡Sets only store hashable items — ints, strings, tuples. Lists and dicts can't be stored in sets. Sets are unordered — iteration order is unpredictable.

Section 09
Hash Maps (dict)
🧠 Mental Model

Python's dict is a hash map. Use it to count frequencies, cache results, build adjacency lists, or map one value to another. The .get(key, default) method is your best friend — no KeyError, returns default if missing.

Safe access with .get()

d.get(key, 0) returns 0 if key doesn't exist instead of raising KeyError. The frequency-counting pattern: count[c] = count.get(c, 0) + 1.

Iterating a dict

d.keys(), d.values(), d.items(). The most common pattern: for key, val in d.items():. In Python 3.7+ dicts maintain insertion order.

defaultdict shortcut

from collections import defaultdict. defaultdict(int) auto-initializes missing keys to 0. defaultdict(list) auto-initializes to [].

Section 10
Tuples
🧠 Mental Model

Tuples are immutable lists. Because they can't change, they're hashable — they can be used as dict keys or stored in sets. Lists are mutable, so they're NOT hashable. This distinction is critical for graph coordinates, cache keys, and more.

Tuples are hashable — lists are not

(1, 2) can be a dict key or set element. [1, 2] cannot — TypeError: unhashable type: 'list'. Use tuples for multi-dimensional coordinates.

Tuple unpacking

x, y = (1, 2) unpacks a tuple. This is why for i, v in enumerate(lst) works — enumerate yields (int, value) tuples.

✅One-element tuple needs a trailing comma: (42,). Without the comma, (42) is just a parenthesized int.

Section 11
Heaps
🧠 Mental Model

Python's heapq implements a min-heap — the smallest element is always at index 0. For a max-heap, negate all values when pushing and negate again when popping. Heaps are used for "k smallest/largest" problems and Dijkstra's algorithm.

heapq operations

heapq.heappush(h, x) O(log n) · heapq.heappop(h) O(log n) · h[0] peek at min O(1) · heapq.heapify(lst) O(n)

Max-heap trick

Negate values: heappush(h, -x). When popping: -heappop(h). This flips the ordering so the largest value is at the top.

Push tuples for custom ordering

heappush(h, (priority, value)). Tuples are compared lexicographically — first by priority, then value. This is the standard pattern for Dijkstra's.

Section 12
Functions
🧠 Mental Model

Functions use def. Nested functions can read outer variables freely (closures). But to reassign a primitive in an outer scope, you need nonlocal. You do NOT need nonlocal to mutate a list — mutation and reassignment are different.

Closures — read for free

Inner functions can read outer variables without any keyword. Only reassigning a primitive (int, str, bool) requires nonlocal.

nonlocal — for reassigning primitives

Without nonlocal, count = count + 1 inside an inner function creates a new local variable, leaving the outer one unchanged. nonlocal count tells Python to use the outer scope's variable.

Mutation vs Reassignment

lst.append(x) is mutation — it modifies the existing object. No nonlocal needed. lst = [1,2,3] is reassignment — replaces the reference. Needs nonlocal.

Section 13
Classes
🧠 Mental Model

Python classes use __init__ as the constructor and require self as the explicit first argument of every method. Unlike Java's implicit this, Python makes the instance parameter visible. All instance variables are set via self.name = value.

self is explicit

Every instance method takes self as its first parameter. When you call obj.method(), Python automatically passes obj as self. You don't pass it manually.

__init__ — constructor

def __init__(self, ...) is called when you create an object. Set instance variables here: self.value = value. Without self., you're creating a local variable that disappears.

Interview: ListNode & TreeNode

Almost every linked list and tree problem uses a simple class. Memorize: class ListNode: def __init__(self, val=0, next=None): self.val=val; self.next=next
